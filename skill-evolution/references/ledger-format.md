# 经验账本条目格式

账本位置：`.claude/skills/skill-evolution/ledgers/<skill>.md`（每个目标 skill 一份）。
账本**只追加**（反思阶段）与**改状态**（固化阶段），不删除历史。

## 单条经验格式

```markdown
### [YYYY-MM-DD] <一句话标题>
- **类型**: 修正 | 补缺 | 固化优点 | 新知识 | 模板摩擦
- **触发**: 对话里可观察到的事实(用户说了什么 / agent 做错或漏了什么)
- **提炼**: 从中得到的、可跨任务复用的一般性结论
- **建议改动**: 落到哪个文件 / 哪个章节，大致怎么改(仅建议，实际改动在固化阶段定)
- **置信**: 高 | 中 | 低(低置信留待固化时人工裁决)
- **状态**: 待固化
```

固化后，把最后一行改为：

```markdown
- **状态**: 已固化(→ CHANGELOG 2026-07-02)
```

## 示例

### [2026-07-02] 项目伤害公式统一用 ATK*(1-DEF/(DEF+K)) 形式
- **类型**: 新知识 / 修正
- **触发**: 用户指出 agent 产出的减法减伤公式「DMG=ATK-DEF」不符合项目规范，应统一用除法减伤，K 由数值定。
- **提炼**: 本项目战斗减伤统一采用 `DMG = ATK * (1 - DEF/(DEF+K))`，combat 只定义此结构、K 交数值。
- **建议改动**: combat-design/references/damage-formulas.md 明确此为项目默认减伤形式；SKILL.md 输出契约「伤害公式体系」加一句默认口径。
- **置信**: 高
- **状态**: 待固化

### [2026-07-02] qa 复盘总漏「配表外键跨表引用」检查
- **类型**: 补缺
- **触发**: 连续两次 qa-review 产出未发现明显的外键指向不存在目标表的问题，用户手动补出。
- **提炼**: qa 的配表校验清单缺「外键目标表存在性」这一独立检查项，容易漏。
- **建议改动**: qa-review/SKILL.md「配表数据校验」清单显式增加一条外键跨表存在性检查步骤。
- **置信**: 中
- **状态**: 待固化

## 使用提示

- 一条经验若同时影响多个 skill，在每个相关 skill 的账本各记一条，互相在标题里注明。
- 「建议改动」写清落点，能大幅降低固化阶段的成本。
- 宁可多记一条低置信观察，也不要在反思阶段直接动 skill。
